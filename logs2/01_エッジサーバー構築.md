# Helix Core コミット/エッジサーバー構築手順

## 概要

Helix Core (Perforce) のサーバー構成の一つである「コミット/エッジサーバー構成」について、コミットサーバーとエッジサーバーの構築手順  
Perforce公式ドキュメントの内容を基に、実際の構築作業に即した形で手順をまとめています
  

### 前提

  * **対象ドキュメント**: Helix Core Server 2020.2
  * **動作確認バージョン**: Helix Core Server(p4d) 2020.2, 2020.1
  * **必要な環境**:
      * サーバー管理用の `super` 権限を持つアカウント
      * コミットサーバー、エッジサーバーとして利用できるサーバー環境をそれぞれ1台ずつ
      * 両サーバー間で通信ができる環境

### サーバーの役割

  * **コミットサーバー (Commit Server)**:
バージョン管理される全ファイルとメタデータ（ファイル履歴、ユーザー、ブランチなど）を保持する中心的なサーバーです。すべての変更は、最終的にこのサーバーにコミットされます。
  * **エッジサーバー (Edge Server)**:
読み取り専用のメタデータコピーと、特定のクライアントが使用するファイルのリビジョンキャッシュを保持します。ユーザーはエッジサーバーに接続し、同期や提出を高速に行うことができます。提出された変更はコミットサーバーに転送されます。

### ディレクトリ構成例

以下の例は、管理上の混乱を避けるため、コミットサーバーとエッジサーバーで同様の構成にすることを推奨します。

  * **コミットサーバー**
    ```
    /parent/p4root/
    ├── journals
    ├── logs
    └── root
    ```
  * **エッジサーバー**
    ```
    /child/p4root/
    ├── journals
    ├── logs
    └── root
    ```

## 構築のポイント

  * **既存サーバーの活用**:
既にコミットサーバーが稼働している場合、既存の設定を利用してセットアップを進めます。新しいコミットサーバー設定を作成しようとするとエラーが発生します。
  * **エッジサーバーの初期状態**:
エッジサーバーは、p4dのインストールが完了した状態で、手順が回ってくるまで特に操作する必要はありません。
  * **SSL証明書**:
SSL通信を行う場合、別途証明書ファイル（`privatekey.txt`と`certificate.txt`）が必要です。これらのファイルがないと、サーバーは起動に失敗します。

## 手順

### 1\. コミットサーバーでの作業

#### 1-1. サービスユーザーアカウントの作成

コミットサーバーとエッジサーバー間の通信に利用する**サービスユーザー**を作成します。このアカウントは、ユーザーが直接ログインして操作するものではありません。

1.  **アカウントの作成**:
    `p4 user ${new_username}` コマンドでエディタを開き、`Type:` を `service` に変更します。コミットサーバー用とエッジサーバー用にそれぞれアカウントを作成してください。

      * 例: `svc_commitedge_parent`（コミットサーバー用）、`svc_commitedge_child`（エッジサーバー用）

2.  **パスワードの設定**:
    `p4 passwd ${username}` コマンドで、各アカウントのパスワードを設定します。

3.  **無制限のグループ設定**:
    タイムアウトしないユーザーグループを作成し、先ほど作成したサービスユーザーを追加します。
    `p4 group no_timeout` を実行し、エディタで以下のように設定します。

    ```
    Group: no_timeout
    Timeout: unlimited
    Users:
      ${commitserver_serviceuser}
      ${edgeserver_serviceuser}
    ```

4.  **`super` 権限の付与**:
    `p4 protect` コマンドでプロテクションテーブルを編集し、両アカウントに `super` 権限を付与します。

    **注意**: 既存の `super user` の記述は絶対に削除しないでください。

    ```
    Protections:
        ...
        super user ${commitserver_serviceuser} * //...
        super user ${edgeserver_serviceuser} * //...
    ```

#### 1-2. サーバー設定の構成

コミットサーバーとエッジサーバーの設定をコミットサーバー側で構成します。

1.  **コミットサーバー設定の編集**:
    `p4 servers` コマンドで既存のサーバーIDを確認し、`p4 server ${servername}` で設定を編集します。

      * `DistributedConfig:` セクションに、以下の設定を追加します。
        ```
        serviceUser=${commitserver_serviceuser}
        monitor=2
        lbr.autocompress=1
        P4TICKETS=.p4tickets
        ```
      * もし `#any` から始まる行がある場合は、その行を個別のサーバー設定にしたい場合は先頭の `#any` を取り除きます。

2.  **エッジサーバー設定の作成**:
    `p4 server -c edge-server ${edge_servername}` コマンドでエッジサーバーの設定を作成します。

      * `ExternalAddress`: エッジサーバーのIPアドレスとポート番号を指定します。
        例: `ssl:${edgeserver_ip}:${useport}`
      * `DistributedConfig:`: 以下を設定します。
        ```
        db.replication=readonly
        lbr.replication=readonly
        lbr.autocompress=1
        rpl.compress=4
        startup.1=pull -i 1
        startup.2=pull -u -i 1
        startup.3=pull -u -i 1
        P4TARGET=ssl:${commit_server_ip}:${useport}
        serviceUser=${edgeserver_serviceuser}
        monitor=1
        journalPrefix=../journals/child
        P4TICKETS=.p4tickets
        P4LOG=../logs/edge_server_log
        ```

#### 1-3. チェックポイントの作成

エッジサーバーの初期データとして使用するため、コミットサーバーのチェックポイントを作成します。エッジサーバーに不要なテーブルを除外するために `-K` フラグを使用します。

```
p4d -r ${commitserver_P4ROOT} -K <LIST OF TABLES> -z -jd edge.ckp
```

`<LIST OF TABLES>` の詳細については、Perforce公式ドキュメントを参照してください。

### 2\. エッジサーバーでの作業

#### 2-1. 初期セットアップ

1.  **ディレクトリ構成の作成**:
    コミットサーバーと同様のディレクトリ構成を作成し、**コミットサーバーで作成したチェックポイントファイル (`edge.ckp.gz`) をエッジサーバーの `${edgeserver_P4ROOT}` に移動**します。

2.  **チェックポイントの復元**:
    チェックポイントファイルからデータを復元します。

    ```
    p4d -r ${edgeserver_P4ROOT} -z -jr edge.ckp.gz
    ```

3.  **サーバーIDの設定**:
    サーバーIDを、コミットサーバーで作成したエッジサーバー設定名に設定します。

    ```
    p4d -r ${edgeserver_P4ROOT} -xD ${edgeserver_name}
    ```

#### 2-2. 通信設定と起動

1.  **SSL証明書の作成**:
    `$P4SSLDIR` のパスをエクスポートし、SSL証明書を作成します。

    ```bash
    mkdir -p ${edgeserver_P4ROOT}/ssl
    chmod 700 ${edgeserver_P4ROOT}/ssl
    export P4SSLDIR=${edgeserver_P4ROOT}/ssl
    p4d -r ${edgeserver_P4ROOT} -p ssl:${edge_server_ip}:${port} -Gc ${P4SSLDIR}
    ```

    **注意**: 証明書作成後、サーバー起動時には `-Gc` オプションは不要です。

2.  **信頼関係の確立**:
    コミットサーバーとの通信を確立するために、信頼ファイルとログインチケットを取得します。

    ```bash
    p4 -E P4TRUST=${edgeserver_P4ROOT}/.p4trust -u ${edgeserver_serviceuser} -p ssl:${commitserver_ip}:${useport} trust
    p4 -E P4TRUST=${edgeserver_P4ROOT}/.p4trust -E P4TICKETS=${edgeserver_P4ROOT}/.p4tickets -u ${edgeserver_serviceuser} -p ssl:${commitserver_ip}:${useport} login
    ```

3.  **エッジサーバーの起動**:
    バックグラウンドでエッジサーバーを起動します。
    **注意**: 同じポートで別のプロセスが動いていないか確認してください。

    ```bash
    p4d -r ${edgeserver_P4ROOT} -p ssl:${edge_server_ip}:${useport} -d
    ```

4.  **起動確認**:
    `p4 info` コマンドを実行し、`Server services: edge-server` と表示されていれば構築成功です。

-----

## トラブルシューティング

  * **起動に失敗する**:

      * SSL証明書（`privatekey.txt`、`certificate.txt`）が正しく作成されているか確認してください。
      * `p4d` のログを確認し、エラーメッセージから原因を特定してください。
      * サーバー設定で指定したディレクトリ（例: `journalPrefix`）が物理的に存在するか確認してください。

  * **同期が始まらない**:

      * エッジサーバーの `.p4tickets` ファイルが、設定で指定した `P4TICKETS` のパスに存在するか確認してください。
      * コミットサーバーとエッジサーバー間で、`trust` および `login` が成功するか再確認してください。
