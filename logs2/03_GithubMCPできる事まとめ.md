# GitHub MCP サーバーの機能まとめ

## できること

### Issues 管理
- Issue の作成・更新・取得
- Issue へのコメント追加
- Issue のラベル追加・削除（既存ラベルのみ）
- Sub-Issue の管理（追加・削除・優先順位変更）
- Copilot への Issue アサイン

### Workflow & Actions
- ワークフローの実行・再実行
- ワークフロー実行結果の確認
- ジョブログの取得
- 失敗したジョブの再実行
- アーティファクトのダウンロード

### リポジトリ情報
- コミット履歴の取得
- ブランチ・ファイル情報の確認
- 認証ユーザー情報の取得

### 検索機能
- ユーザー・リポジトリ・Issues・コードの検索

## できないこと

### リポジトリ管理
- 新しいリポジトリの作成
- ファイルの作成・編集・削除
- ブランチの作成・削除
- プルリクエストの作成・管理

### ラベル・設定管理
- 新しいラベルの作成
- ラベル自体の編集・削除
- リポジトリ設定の変更

### 高度な機能
- セキュリティアラートの管理
- Wiki の編集
- Discussions の管理
- Gist の管理

## 役割分担

### 🔧 VS Code内のCopilot（ローカル環境）
- **ソースコードの編集・修正**
- **ファイルの作成・削除**
- **リファクタリング**
- **コード補完・生成**
- **デバッグ支援**
- **ローカルでのテスト実行**

### GitHub MCP サーバー
- **GitHub Actions の実行**
- **Issue・PR の管理**
- **リポジトリ情報の取得**
- **リモートでの作業状況確認**
- **CI/CD パイプラインの監視**

## 開発フロー例
1. VS Code Copilot でコード修正 ↓
2. ローカルでテスト ↓
3. Git push ↓
4. GitHub MCP でActions実行 ↓
5. GitHub MCP で結果確認 ↓
5. GitHub MCP でIssue更新

## まとめ

**要約**: 「参照・実行・Issue管理」は強力ですが、「作成・編集・設定変更」系の機能は制限されている状況です。

**住み分け**:
- **「作る・直す」= VS Code Copilot**
- **「動かす・管理する」= GitHub MCP**

両方を組み合わせることで、ローカル開発からリモート管理まで一貫したワークフローが実現できます。
